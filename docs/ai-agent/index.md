# AI Agent 工程

> **核心概念：AI Agent 工程构建的是一个由模型参与决策、由软件负责执行和约束的闭环系统。**

大模型可以理解自然语言和非结构化信息，软件系统则擅长执行确定的规则、保存状态和控制风险。AI Agent 工程把两者连接起来，使模型能够在软件提供的边界内获取信息、调用工具并推进任务。

## 决策闭环与工程边界

一个 Agent 系统可以从两个层次理解。

内部是不断运行的**决策闭环**：

```text
目标 + 当前状态
       ↓
模型判断下一步
       ↓
工具执行动作
       ↓
结果成为新的状态
       └── 继续决策，直到完成或终止
```

闭环中有四个相互连接的核心概念：

- **目标**说明系统要完成什么，并提供结束条件。
- **决策**由模型根据上下文和当前状态选择下一步。
- **行动**通过工具读取信息或改变外部系统。
- **反馈**把工具结果写回状态，成为下一轮决策的依据。

闭环之外是软件建立的**工程边界**。权限和人工审批限制 Agent 能做什么；评测和可观测性判断它是否做对；超时、重试和终止条件防止任务失控；性能与成本治理保证系统能够长期运行。

因此，AI Agent 工程的难点不只是让模型完成一次演示，而是同时设计好决策闭环和工程边界。哪些步骤交给模型，哪些仍由程序控制，是贯穿整个知识体系的核心问题。

## 一分钟复述

AI Agent 工程把大模型的判断能力接入软件系统。Agent 内部通过“目标、决策、行动、反馈”形成循环，根据工具结果不断推进任务；外部则由权限、评测、可观测性和故障控制建立边界。模型负责处理无法完全预先写死的决策，软件负责可靠执行和风险控制。衡量一个 Agent 系统的标准，不是它有多自主，而是它能否稳定、安全地完成业务目标。
