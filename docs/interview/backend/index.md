# 后端面试题

后端面试既会考察 Java、JVM、事务、索引和缓存等基础概念，也会通过超时、并发、消息重复和系统过载等场景判断工程能力。

题库按常见面试主题组织，便于集中复习；[后端工程]({{ site.baseurl }}/docs/backend/)正文仍按一次业务结果怎样产生、保存和跨越故障的主线组织。两种顺序服务不同目的。

## Java 技术栈

Java 后端问题从语言语义、集合与并发进入 Spring、JVM、垃圾回收和生产排障。Spring 只保留 IoC、Bean、AOP、事务和 Spring Boot 自动配置等高频机制，不展开注解清单和 Spring MVC 组件细节。

[Java 技术栈常见面试题](java-jvm/)

## 数据库

数据库问题通常从事务、隔离级别、索引和锁开始，再进入慢查询与并发更新，页面末尾附 MySQL 专项。回答时要区分数据库保证、具体产品实现、应用业务规则和跨服务问题，不能把本地事务的能力扩大到整个分布式系统。

[数据库常见面试题](database/)

## 缓存

常见问题包括缓存的适用场景、Cache Aside、缓存一致性、穿透、击穿和雪崩，页面末尾附 Redis 专项。重点不仅是术语区别，还包括缓存故障时怎样保护数据库。

[缓存常见面试题](cache/)

## 消息与异步任务

常见问题包括异步、削峰、消息丢失与重复、投递语义、消费幂等、顺序、积压和死信，页面末尾附 Kafka 的 Partition、消费组、副本与事务语义。

[消息队列与 Kafka 常见面试题](message-queue/)

## 分布式系统

常见问题包括 CAP、幂等、超时与重试、服务治理、分布式事务、分布式锁和限流。回答时应说明故障窗口、权威状态与恢复依据，而不只是背诵方案名称。

[分布式系统常见面试题](distributed-system/)

## 性能与生产运行

常见问题包括吞吐量、尾部延迟、连接池、背压、指标、日志、Trace、事故止损、灰度发布和兼容迁移。回答要从用户结果走到瓶颈、止损、恢复和长期改进。

[性能与生产运行常见面试题](performance-production/)

## 后端安全与鉴权

常见问题包括认证与授权、Session 与 JWT、OAuth 与 OIDC、RBAC 与 ABAC、多租户隔离、SQL 注入、XSS、CSRF、CORS 和接口重放防护。

[后端安全与鉴权常见面试题](security-auth/)

## API 与应用层

常见问题包括 REST 资源、HTTP 方法与状态码、入参与业务校验、接口幂等、分页、异步接口、长任务状态和版本兼容。

[API 与应用层常见面试题](api-application/)

## 大模型应用后端

常见问题包括流式事件、Tool Call 执行、Agent 持久状态、Worker 恢复、Token 与并发预算、可观测性和组合发布。

这些问题建立在通用后端能力之上。模型会提出候选动作，真正的权限、状态、幂等和故障恢复仍由应用后端实施。

[大模型应用后端常见面试题](llm-application-backend/)
