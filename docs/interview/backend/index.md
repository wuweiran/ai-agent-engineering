# 后端面试题

后端面试既会考察事务、索引和缓存等基础概念，也会通过超时、并发、消息重复和系统过载等场景判断工程能力。

题库按常见面试主题组织，便于集中复习；[后端工程]({{ site.baseurl }}/docs/backend/)正文仍按一次业务结果怎样产生、保存和跨越故障的主线组织。两种顺序服务不同目的。

## 数据库

数据库问题通常从事务、隔离级别、索引和锁开始，再进入慢查询与并发更新，页面末尾附 MySQL 专项。回答时要区分数据库保证、具体产品实现、应用业务规则和跨服务问题，不能把本地事务的能力扩大到整个分布式系统。

[数据库常见面试题](database/)

## 缓存

常见问题包括缓存的适用场景、Cache Aside、缓存一致性、穿透、击穿和雪崩，页面末尾附 Redis 专项。重点不仅是术语区别，还包括缓存故障时怎样保护数据库。

[缓存常见面试题](cache/)

## 消息与异步任务

常见问题包括为什么使用消息队列、消息丢失与重复、投递语义、消费幂等、顺序、积压、死信，以及 Worker 怎样在重启后继续任务。

## 分布式系统

常见问题包括幂等、超时与重试、分布式事务、Outbox、最终一致性、限流、熔断、降级和分布式锁。回答时应说明故障窗口和结果是否确定，而不只是背诵组件定义。

## 性能与生产运行

常见问题包括吞吐量与延迟、连接池、排队、背压、系统观测、事故处理、灰度发布和数据库迁移。面试官通常希望看到从现象到瓶颈、从止损到恢复的判断过程。

## 大模型应用后端

常见问题包括 SSE 与 WebSocket 的选择、模型输出与业务事实的边界、Tool Call 的执行、Agent 状态持久化，以及模型或工具调用超时后的恢复。

这些问题建立在通用后端能力之上。模型会提出候选动作，真正的权限、状态、幂等和故障恢复仍由应用后端实施。
