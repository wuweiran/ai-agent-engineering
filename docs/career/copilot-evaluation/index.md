# Copilot Evaluation 与 Golden Set

## 项目是什么

与 Copilot 功能开发并行，参与建立 Golden Set 和回归评测流程。模型、Prompt、Context、内容提取和工具版本都会影响结果，开发人员手工试几个邮箱无法判断一次修改是否真正改善。

这个项目把真实用户任务保存成可重复运行的样本，并比较不同 Copilot 版本在回答、引用、工具和动作上的变化。

## 评测对象

Golden Set 不只是问题和标准答案。样本还需要对应的邮箱或附件数据、允许使用的 Context、必须覆盖的事实、不能出现的结论，以及邮箱整理中的预期动作和禁止动作。

评测通常同时检查：

- 回答是否解决用户问题；
- 关键事实是否完整且受到证据支持；
- 引用是否对应原文；
- 工具和参数是否正确；
- 动作是否越权或影响错误资源；
- 延迟、Token 和成本是否可以接受。

## 主要工作

主要工作包括 Golden Set 数据准备、评测执行、结果保存、评分与基线比较。Schema、资源 ID、引用位置和动作范围等确定结果由程序判断；开放语义使用评分模型，并通过人工抽查校准。

这个项目可以深入说明 Agent 评测、严重失败、Trace、第一次偏离、消融实验和发布门槛。它也包含一项长期工作：功能和数据变化后，Golden Set 本身需要更新和版本化，否则评测会逐渐失真。

## 后续可以展开的文档

- Golden Set：样本来源、数据快照、标签、版本和权限；
- 评测执行：模型、Prompt、Context、工具版本和并发运行；
- Grader：确定性检查、模型评分、人工校准和一致性；
- 指标与报告：完整性、忠实度、引用、任务完成和严重失败；
- 发布闭环：回归、灰度、线上失败回收和停止条件；
- 评测平台部署：任务队列、计算资源、存储、容量和成本。
